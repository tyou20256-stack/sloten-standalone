## 2. Golden Set 設計

- **サイズ**: 200件（MVP）→ 段階的に500件
- **構成比**:
  - 入出金 60件 (30%)
  - ボーナス・キャンペーン 40件 (20%)
  - アカウント・KYC 40件 (20%)
  - コンプライアンス必須エスカレ 30件 (15%)
  - 雑談・範囲外 30件 (15%)
- **構築手順**:
  1. 既存 ai_logs から頻出意図 TOP30 を抽出
  2. 各意図に対し「模範回答」「必須含有キーワード」「NGワード」「期待 KB ドキュメント ID」「期待エスカレーションフラグ」をオペレーターが記入
  3. D1 テーブル `golden_set` として格納
- **使い方**: プロンプト/KB 変更時に CI で全件再生成 → 後述メトリクス自動判定 → 閾値割れでマージブロック
- **数値目標**: Grounding 率 現状不明 → **85% 以上**、Compliance 違反率 不明 → **0.5% 未満**

---

## 3. 自動評価メトリクス 4 つ

1. **Keyword Inclusion Score** — 模範回答の必須キーワード含有率。正規表現で判定。目標 **90%**
2. **KB Citation Hit Rate** — 回答が KB ドキュメント ID 参照を含むか、また embedding 類似度で KB 由来判定（cosine > 0.72）。目標 **80%**
3. **LLM-as-Judge（Claude Haiku 別モデル）** — 「事実性」「丁寧さ」「範囲内か」の 3 軸を 1-5 で採点。コスト $0.001/件。目標 **平均 4.3 以上**
4. **NG Pattern Detection** — 正規表現で「確実に勝てる」「必ず還元」「入金してください」等の景表法・不当誘引ワードを検知。目標 **検知率 0 件/日**

## 4. 運用者フィードバック UI 改善

- **ワンクリック評価**: ai_logs 詳細画面に「👍👎」2 択＋任意コメント。3 秒で完了
- **日次レビューキュー**: LLM-as-Judge スコアが Bottom 20% のログのみを「要レビュー」タブに自動集約。全件見る必要を排除
- **インセンティブ**: 週次で「フィードバック件数」を管理画面に表示、KPI 化（目標: 日次 10 件以上）
- **フィードバック → Golden Set 昇格**: 👎 ログは修正回答を付けて Golden Set に追加するフローをボタン 1 つで実行

---

## 5. Regression Prevention

- **CI/CD 統合**: GitHub Actions でプロンプト/KB 変更 PR 時に `npm run eval:golden` を必須チェック化
- **閾値ゲート**: Keyword Inclusion < 88%、LLM-Judge < 4.2、NG 検知 > 0 のいずれかでマージブロック
- **Shadow deploy**: 新プロンプトは 7 日間、既存プロンプトと並走スコアリング → 有意差（McNemar 検定 p<0.05）で採用判定
- **A/B プロンプト決着**: 4 プロンプト × Golden Set 200 件で一括採点、週次で勝者固定・敗者ローテ
