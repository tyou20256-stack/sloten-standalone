# Support Responder 視点: AI/人間境界とエスカレーション判断

**エージェント**: Support Responder (CS 20 年視点)
**視点**: AI が答えるべきでない質問 / 依存症対応 / 過剰約束防止 / 炎上防止
**総合方針**: 「正答できる」と「答えるべき」は別。Accuracy 向上 ≠ AI に全部任せること。

---

## 1. AI が答えるべきでない質問 TOP 7

| # | 類型 | 検出シグナル | 例 |
|---|---|---|---|
| 1 | 金銭トラブル | 「返金」「出金できない」「引かれた」「消えた」 | 「入金したのに反映されない」 |
| 2 | 依存症兆候 | 「やめたい」「借金」「生活費」「最後」 | 「もう全部負けた」 |
| 3 | 個別事情の減免要求 | 「死亡」「入院」「離婚」「失業」「うつ」 | 「親が亡くなって払えない」 |

| 5 | 本人確認・アカウント凍結 | 「凍結」「ログインできない」「KYC」「本人確認」 | 「アカウントが使えない」 |
| 6 | 未成年・第三者利用疑い | 「18歳」「未成年」「子供が」「勝手に」 | 「息子が勝手に登録」 |
| 7 | 個人情報変更・退会 | 「退会」「削除」「個人情報」「GDPR」 | 「登録情報を全部消して」 |

## 2. エスカレーション判断の実装案

三層 AND/OR で precision を確保:

```typescript
shouldEscalate(msg, history) {
  const hardKW  = /返金|出金できない|訴え|弁護士|消費者センター|凍結|未成年|退会|削除/;
  const softKW  = /やめたい|借金|生活費|親|死亡|入院|離婚|うつ|最後/;
  const angerKW = /ふざけ|詐欺|最悪|二度と|炎上|SNS|晒す/;
  const repeat  = history.filter(m => m.user).slice(-3)
                    .some(m => similarity(m.text, msg) > 0.7);  // 同質問 3 回

  if (hardKW.test(msg)) return { escalate: true, reason: 'hard' };
  if (angerKW.test(msg) && sentiment(msg) < -0.5) return { escalate: true, reason: 'anger' };
  if (softKW.test(msg) && sentiment(msg) < -0.3) return { escalate: true, reason: 'rg_soft' };
  if (repeat) return { escalate: true, reason: 'deadloop' };  // dead-loop 防止
  return { escalate: false };
}
```

## 3. 過剰約束防止 — 禁止ワード (AI 出力フィルタ)

```
禁止:「必ず」「絶対」「確実に」「100%」「保証」「〜円もらえます」
     「24時間以内に」「即時」「すぐに反映」「当選します」「勝てます」
置換:「通常は」「目安として」「〜の場合がございます」
```

景表法優良誤認対策として「勝率」「還元率」の具体数値言及も AI からは禁止。

## 4. 依存症兆候への必須対応 (RG 責任)

検出したら **即ハードコード返答 → 人間エスカレ**:

> 「ご状況お察しいたします。まず専門の相談窓口をご案内させてください。
>
> **ギャンブル依存症 相談窓口 (無料)**
>   生活支援センター: 0570-069-200
>   GA 日本: 0172-58-8199
>
> またご希望があれば、当サイトの自己規制 (入金上限/利用停止) 設定もすぐ承ります。担当者におつなぎします。」

AI 生成を通さない **固定文言 + 自己規制機能導線 + 人間引継** が必須。

## 5. 怒り顧客 NG 集

| NG 応答 | 炎上理由 | 代替 |
|---------|---------|------|
| 「規約に記載の通りです」 | 突き放し | 「ご不快な思いをおかけし申し訳ございません。状況を担当が確認します」 |
| 「システム上可能です」 | 事務的 | 「確かに分かりづらい表示でした」 |

| 絵文字・「！」連発 | 軽薄 | プレーンな敬語 |
| 即座の再質問 | 火に油 | 先に謝罪→要約→事実確認 |

## 6. Escalation Rate 目標

- **理想 25-35%** (業界ベンチ: カジノ CS 30-40%)
- **15% 未満** = AI が抱え込み過ぎ (苦情化リスク)
- **45% 超** = AI 機能不全 (コスト負け)
- **現状 open 528/2721 ≒ 19%** は **やや低め**。RG/金銭系の取りこぼしを疑う

---

## 7. AI が正答できても人間に回すケース

1. **VIP/高額入金者** (CRM tag = vip) — 関係性維持の方が ROI 高い
2. **初回入金前の不安質問** — コンバージョン直結。人の温度が必要
3. **長文 (200 字超) の相談** — 要約ミスで炎上リスク
4. **深夜帯 + ネガティブ感情** — 依存/衝動性サイン
5. **3 回目以降の再問合せ** — AI で解決しなかった既成事実
6. **景表法・特商法に触れる質問** — 法務確認必須
7. **「SNS に書く」示唆** — 即シニア対応 (CPA 2-3 倍損失予防)

## 実装優先度

| Phase | 内容 |
|-------|------|
| **Phase 1 (即対応)** | §1 hardKW 辞書 + §4 RG 固定文言 + §3 禁止ワード出力フィルタ |
| **Phase 2 (2 週)** | §2 sentiment + dead-loop 検出、§7 の VIP/深夜ルーティング |
| **Phase 3 (1 ヶ月)** | escalation rate ダッシュボード監視 → 25-35% 帯に調整 |

sloten-standalone の既存「担当者におつなぎします」ブランケットを、**理由コード付きエスカレ** (hard / anger / rg_soft / deadloop / vip) に置換することで、人間側の初動速度も上がります。
