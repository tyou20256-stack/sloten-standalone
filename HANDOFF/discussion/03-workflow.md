## 3. エンドツーエンドフロー分析 (入金→残高反映)

| Step | 実施者 | 作業 | 所要時間 | 待ち時間 |
|---|---|---|---|---|
| 1 | 顧客 | widget で入金申請・金額入力 | 30 秒 | 0 |
| 2 | GAS bot | 申請受付・申請ID発行 | 2 秒 | 0 |
| 3 | 顧客 | PayPay アプリで送金実行 | 60 秒 | 0 |
| 4 | 顧客 | スクショ取得・widget に添付 | 30 秒 | 0 |
| 5 | GAS | スプシに pending 行追加 | 3 秒 | 0 |
| 6 | **(待機)** | staff 確認待ちキュー | 0 | **5〜120 分** (シフト/混雑依存) |
| 7 | staff | スプシを開く・新規行確認 | 15 秒 | 0 |
| 8 | staff | スクショ目視・金額/送金者名突合 | 90 秒 | 0 |
| 9 | staff | スプシに OK ステータス記入 | 30 秒 | 0 |
| 10 | staff | 残高反映 bot/API を発火 | 20 秒 | 0 |
| 11 | GAS | 顧客に完了通知送信 | 2 秒 | 0 |

**エンドツーエンド合計**: 正味 4 分程度 + 待機 5〜120 分
**ボトルネック**: Step 6 (待機キュー)。ピーク時/深夜は 2 時間超待ちも発生推定。これが **顧客体験の 95% を劣化させる単一要因**。

## 4. スプレッドシート運用のリスク評価

| リスク | 現状影響度 | 発生頻度 (推定) |
|---|---|---|
| 並行編集 conflict | 金額誤上書き・重複付与 | 月 2〜5 件 |
| 見落とし (スクロール外の行) | 未処理放置・顧客クレーム | 月 3〜10 件 |
| 誤記入 (行ズレ・金額タイポ) | 過少/過剰付与 | 月 2〜4 件 |
| 監査証跡欠如 | 誰がいつ変更したか追跡不可 | 常態 |
| 退勤後の放置 | 深夜/早朝の申請が朝まで滞留 | 毎日 |
| 共有リンク漏洩 | 顧客 PII/取引額流出 | 潜在リスク |

**代替案比較**:

- **案A: スプシを残して API 化** (GAS Web App + 構造化 sheet)
  - 工数: 2〜3 週間 / コスト低 / 既存運用踏襲で移行容易
  - 欠点: 根本リスク (行ズレ・並行編集) は残存。中期的には負債化

- **案B: D1 + 専用 admin UI へ移行** (推奨)
  - 工数: 6〜8 週間 / コスト中 / 監査ログ・権限制御・楽観ロック標準装備
  - トランザクション保証、row-level lock で重複付与物理的に不可
  - スプシは read-only export として残す (スタッフ心理的安全)

**推奨**: Phase 1 で案A (API 化) → Phase 3 で案B (D1 移行)。段階移行でリスク分散。

## 5. 改善ロードマップ

### Phase 1 (0〜1 ヶ月): Quick wins 消化

- PayPay スクショ → OCR (Google Vision / Gemini Vision) で金額・送金者名抽出
- GAS に突合ロジック実装 (申請金額 ±0 円完全一致で自動 approve)
- 残高反映 Webhook 化 (スプシ記入と同時に API 発火)
- ボーナスコード有効性の自動検証 (DB 照会)
- 日次サマリの自動生成 (GAS scheduled trigger)

**期待削減**: 約 314 h/月 = **人件費 ¥565,200/月削減**

### Phase 2 (1〜3 ヶ月): 主要自動化

- OCR 信頼度スコアに基づく自動/手動分岐 (>95% 自動、それ以下は staff queue)
- widget bot の FAQ 拡充・RAG 導入でエスカレーション率半減
- トラブル対応テンプレート + 自動返信下書き生成 (staff は承認のみ)
- 出金審査の自動判定 (KYC/上限チェック)

**期待削減**: 追加 180 h/月 = **累計 ¥890,000/月削減**

### Phase 3 (3〜6 ヶ月): 抜本改善

- スプレッドシート → D1 + Cloudflare Workers admin UI 移行
- 監査ログ・RBAC・楽観ロック完備
- 不正検知 ML モデル導入 (取引パターン異常検知)
- 経営 dashboard (Grafana) 自動生成

**期待削減**: 追加 120 h/月 = **累計 ¥1,100,000/月削減**

**3 フェーズ累計投資**: 約 ¥2,500,000〜4,000,000 (外注込)。**6 ヶ月で回収、年間 ¥13M 以上の削減効果**。

**総括**: Quick wins だけで月 300h 超の削減余地あり。最優先は **OCR + 突合自動化 + Webhook 化** の 3 点セット。スプシ運用は短期 API 化で延命、中期で D1 移行が望ましい。待機時間 (平均 45 分) の解消が顧客体験改善の最大レバー。
