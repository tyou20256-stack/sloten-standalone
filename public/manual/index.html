1. 概要

sloten-standalone は Cloudflare Workers + D1 で動作する、スロット天国のカスタマーサポート統合プラットフォームです。

機能カテゴリ

カテゴリ内容
対話顧客 Widget + Operator UI + Bot Flow (109 steps)
ボーナスコード25 種の検出 + 自動応答 + GAS スプシ記録
ナレッジFAQ 47 + knowledge_sources 23 + chunks 56 (FTS5 BM25)
AI 回答Gemini 2.5 Flash Lite + RAG + 11 段階 pipeline
安全装置escalation gate / 過剰約束フィルタ / 依存症対応 / PII masker
評価基盤Golden Set 195 / Shadow mode / LLM-as-Judge
Hybrid RAGBM25 + Vectorize + RRF fusion (opt-in)
対象読者: スロット天国カスタマーサポートの運用者・管理者。ITエンジニア向けではありません。

5.3 新規ボーナスコード追加 (Generic Handler 対応)

  1. + 新規作成 ボタンをクリック
  2. type_key: 英小文字/数字/アンダースコア (例: summer2026)
  3. 表示名: 運用内部の表示名 (例: 夏祭りキャンペーン)
  4. 受付コード: カンマ or 改行で複数可 (例: 夏祭り, summer)
  5. 照合方法: 「大文字小文字を区別しない」(推奨) or 「完全一致」
  6. 成功メッセージ本文: 顧客に送る応答テキスト (改行可)
  7. 成功メッセージ選択肢: 動的フォームで +/- ボタンで追加
    • タイトル: 表示する文字 (例: ↩️ メインメニューに戻る)
    • 値: 遷移先ステップ ID (例: welcome_message)
  8. GAS type: BC_ 固定 + 任意 suffix (例: 夏祭り) — 空欄なら GAS 記録なし
  9. 受付後にオペレーター転送: チェック時はコード受付後に人間に引き継ぎ
  10. 作成 ボタンで保存
v9.4 Generic Handler: GAS コード変更なしで動作します。Google Spreadsheet に BC_夏祭り タブを作成しておけば、自動で 4 列標準フォーマットで記録されます。

5.4 既存ボーナスコード編集

行の「編集」→ 同じフォームで変更 → 保存。

11. FAQ 候補

過去の会話から自動抽出された FAQ 候補。

毎週日曜 18:00 UTC (月曜 03:00 JST) に extract-faqs-from-messages.mjs cron が実行され、新規候補が追加されます。

レビュー操作

  1. 行の 承認 → FAQ 本体に昇格、AI 回答で使用開始
  2. 行の 却下 → 候補から除外
  3. 行の 編集 → 質問・回答を修正してから承認
  4. 一括選択 → 複数選択して一括承認/却下
Phase 2b の Silver 層: 現状 606 候補のうち 587 が rejected。次回から FAQ 候補クラスタ で意味的重複を ~80 cluster に圧縮して提示できます。

12. ボットフロー

顧客の最初のメッセージから人間エスカレまでの対話ツリー。sloten-main が 109 ステップで全カスタマージャーニーをカバー。

ステップの種類

type挙動
messageテキストを送信 → 次ステップへ
selectボタン付きメニュー表示、ユーザー選択を待つ
input自由テキスト入力を var に捕捉 (validate 正規表現可)
webhookGAS or 外部 API を POST、応答を次ステップへ
handoff会話を status=open にして人間にエスカレ

主な操作

  1. sloten-main 行の「編集」を開く
  2. JSON エディタで steps 配列を編集
  3. 保存
危険: sloten-main の構造を壊すと全顧客対応が止まります。scripts/convert-agentbot-messages.mjs で再生成する CLI 運用を推奨。

18. Shadow Mode Phase 2a

18.1 目的

ユーザーに見せず候補プロンプトを並列実行してログだけ取る。2-3 本の新 prompt を本番顧客の質問に当てて比較評価。

18.2 画面

/admin/#shadow-settings
☐ Shadow mode 有効 (ai.shadow_mode.enabled) Shadow prompt IDs (カンマ区切り、最大 2 本): [ ] [ 💾 保存 ] 利用可能なプロンプト: id │ 名前 │ active │ weight 1 │ default-A-detailed │ ✓ │ 50 2 │ default-B-concise │ ✓ │ 50 5 │ default-C-tiered │ ✓ │ 0

18.3 有効化手順

  1. Shadow mode 有効 にチェック
  2. Shadow prompt IDs に評価したい prompt ID をカンマ区切りで入力 (例: 5)
  3. 💾 保存
  4. 以降、ユーザー応答は primary prompt (A or B) のまま、候補 prompt 5 が ai_logs に is_shadow=1 で並列記録される
  5. 数日後、Golden Set eval で比較して勝敗判定
コスト: LLM 呼び出しが 2-3 倍に。1 prompt shadow だと 2x, 2 prompt で 3x。Gemini Flash Lite なので月 $1 未満だが OFF がデフォルト。

19.3 初回セットアップ手順

  1. 📤 KB chunks を reindex クリック
  2. 確認ダイアログで Yes
  3. 56 chunks が Workers AI で embed → Vectorize に push (約 30 秒)

  5. 🔍 Query テスト で任意のクエリを入れてヒット具合を確認
  6. 問題なければ 🟢 Hybrid ON → feature flag ON
  7. 以降、全 AI 回答が hybrid_rrf retrieval 経由
